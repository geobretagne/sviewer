sViewer — extensions

Extensions extend sViewer to load non-GeoJSON sources via ?geojson=. Each extension is a self-contained adapter.js activated in customConfig.js.

CSV adapters: ['csv']
Loads comma or semicolon-separated files and converts them to map features. Geometry is auto-detected from column names — no configuration required for standard files.
Capabilities
  • Auto-detects latitude/longitude columns (and common variants: lat, lon, lng)
  • Auto-detects geometry column containing GeoJSON strings
  • Comma and semicolon separators — auto-detected from first row
  • Quoted fields containing commas or line breaks
  • UTF-8 BOM stripped automatically
  • Invalid rows (missing or non-numeric coordinates) skipped silently
URL hints
HintExampleEffect
_format=csv…&_format=csvForce CSV adapter for extension-less URLs
_collat_collat=latOverride latitude column name
_collon_collon=lngOverride longitude column name
_labelcol_labelcol=nameColumn used as feature label
Run checks → Requires CORS · UTF-8 encoding URL match: .csv extension or _format=csv
Grist adapters: ['grist']
Loads data from the Grist records API and renders it as map features. Works both as a standalone ?geojson= URL and as a Grist widget embedded in a document.
Capabilities
  • Auto-detects geometry column containing GeoJSON strings (Point, LineString, Polygon)
  • Auto-detects latitude/longitude column pair
  • Supports GeoJSON, lat/lon, lat,lon text, lon,lat text, and WKT geometry modes
  • Invalid rows (unparseable geometry) skipped silently
  • Full Grist widget with configuration panel, style, sharing, and real-time sync
URL hints
HintExampleEffect
_geommode_geommode=latlonForce geometry mode (geojson · latlon · latlon_str · lonlat_str · wkt)
_geomcol_geomcol=shapeOverride geometry column name
_collat_collat=latOverride latitude column name
_collon_collon=lngOverride longitude column name
_labelcol_labelcol=nameColumn used as feature label
Run checks → Requires CORS · Public document (no API key) URL match: /api/docs/{id}/tables/{table}/records
Sample adapters: ['sample']
Annotated reference extension for extension authors. Loads a flat JSON array [{latitude, longitude, …}, …] and converts it to map features. Read ext/sample/adapter.js as a starting point for new extensions.
Capabilities
  • Flat JSON array input — no envelope, no nested structure
  • Auto-detects latitude/longitude columns
  • Custom column names via _collat/_collon hints
  • Invalid rows skipped silently
URL hints

_collat_collat=latOverride latitude column name
_collon_collon=lngOverride longitude column name
_labelcol_labelcol=nameColumn used as feature label
Run checks → Reference implementation — not for production use URL match: _format=sample only
